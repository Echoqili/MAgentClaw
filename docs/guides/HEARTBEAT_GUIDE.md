# 心跳机制（Heartbeat）使用指南

## 概述

心跳机制是 MAgentClaw v1.3.0 的核心功能之一，用于实现 Agent 的周期性任务调度和状态监控。参考了 OpenClaw 项目的设计理念，提供了灵活、可靠的任务管理方案。

## 核心特性

- ✅ **周期性任务调度** - 支持自定义间隔的周期性任务
- ✅ **HEARTBEAT.md 文件格式** - 人类可读的任务配置文件
- ✅ **心跳抑制逻辑** - 避免任务重复执行
- ✅ **错误处理和重试** - 自动重试失败任务
- ✅ **状态监控** - 实时跟踪任务执行状态
- ✅ **执行历史** - 记录所有任务执行历史
- ✅ **统计分析** - 提供任务执行统计信息

## 快速开始

### 1. 创建 HEARTBEAT.md 文件

在工作空间目录下创建 `HEARTBEAT.md` 文件：

```markdown
# Heartbeat Tasks

## Task: health-check
- Interval: 60
- Command: check-health --service=all
- Enabled: true

## Task: data-sync
- Interval: 300
- Command: sync-data --source=db --target=cache
- Enabled: true

## Task: cleanup
- Interval: 3600
- Command: cleanup-temp-files
- Enabled: false
```

### 2. 使用 HeartbeatManager

```python
from pathlib import Path
from maagentclaw.managers.heartbeat_manager import (
    HeartbeatManager,
    HeartbeatConfig
)

# 创建工作空间路径
workspace_path = Path("./workspaces/my-agent")

# 创建配置
config = HeartbeatConfig(
    enabled=True,
    interval=60,  # 心跳检查间隔（秒）
    max_retries=3,
    retry_delay=5,
    timeout=300,
    suppress_duplicates=True
)

# 创建管理器
manager = HeartbeatManager(
    workspace_path,
    config,
    execution_callback=your_execution_function
)

# 启动心跳
await manager.start()

# ... 运行中 ...

# 停止心跳
await manager.stop()
```

## API 参考

### HeartbeatConfig

心跳配置类：

```python
@dataclass
class HeartbeatConfig:
    enabled: bool = True           # 是否启用心跳
    interval: int = 60             # 心跳间隔（秒）
    max_retries: int = 3           # 最大重试次数
    retry_delay: int = 5           # 重试延迟（秒）
    timeout: int = 300             # 任务超时（秒）
    suppress_duplicates: bool = True  # 抑制重复执行
    log_level: str = "INFO"        # 日志级别
```

### HeartbeatTask

心跳任务类：

```python
@dataclass
class HeartbeatTask:
    name: str                      # 任务名称
    interval: int                  # 执行间隔（秒）
    command: str                   # 要执行的命令
    enabled: bool = True           # 是否启用
    last_run: Optional[datetime]   # 上次执行时间
    next_run: Optional[datetime]   # 下次执行时间
    status: TaskStatus             # 任务状态
    execution_count: int = 0       # 执行次数
    failure_count: int = 0         # 失败次数
    last_error: Optional[str]      # 最后错误信息
    last_duration: Optional[float] # 最后执行时长
    metadata: Dict[str, Any]       # 元数据
```

## 心跳抑制逻辑

心跳抑制机制防止任务在短时间内重复执行：

1. **运行中抑制** - 如果任务正在运行（status=RUNNING），不会再次执行
2. **时间间隔抑制** - 距离上次执行时间少于间隔的 50%，不会执行
3. **配置开关** - 通过 `suppress_duplicates` 配置项控制

```python
config = HeartbeatConfig(
    suppress_duplicates=True  # 启用抑制
)
```

## 错误处理和重试

### 自动重试

任务执行失败时会自动重试：

```python
config = HeartbeatConfig(
    max_retries=3,    # 最多重试 3 次
    retry_delay=5     # 每次重试间隔 5 秒
)
```

### 重试策略

1. 第一次失败：等待 5 秒后重试
2. 第二次失败：等待 5 秒后重试
3. 第三次失败：等待正常间隔时间

### 错误记录

所有错误都会被记录：

```python
task_status = manager.get_task_status("task-name")
print(task_status["last_error"])      # 最后错误信息
print(task_status["failure_count"])   # 失败次数
```

## 执行回调

执行回调是实际执行任务的函数：

```python
async def execute_task(command: str, metadata: dict):
    """执行任务的回调函数"""
    print(f"执行任务：{command}")
    
    # 这里实现具体的执行逻辑
    # 可以是：
    # - 调用技能
    # - 执行命令
    # - HTTP 请求
    # - 数据库操作
    # 等
    
    return result

# 创建管理器时传入回调
manager = HeartbeatManager(
    workspace_path,
    config,
    execution_callback=execute_task
)
```

## 监控和告警

### 实时监控

```python
# 获取所有运行中的任务
running_tasks = [
    task for task in manager.get_all_tasks()
    if task["status"] == "running"
]

# 获取所有失败的任务
failed_tasks = [
    task for task in manager.get_all_tasks()
    if task["status"] == "failed"
]
```

### 统计信息

```python
stats = manager.get_statistics()

print(f"总任务数：{stats['total_tasks']}")
print(f"启用任务：{stats['enabled_tasks']}")
print(f"运行中：{stats['running_tasks']}")
print(f"失败任务：{stats['failed_tasks']}")
print(f"总执行：{stats['total_executions']}")
print(f"成功率：{stats['success_rate']:.1f}%")
```

### 告警示例

```python
async def monitor_heartbeat(manager):
    """监控心跳并发送告警"""
    while True:
        stats = manager.get_statistics()
        
        # 检查失败率
        if stats['success_rate'] < 90:
            print(f"⚠️  警告：成功率低于 90% ({stats['success_rate']:.1f}%)")
        
        # 检查失败任务
        failed_tasks = [
            task for task in manager.get_all_tasks()
            if task['status'] == 'failed'
        ]
        
        for task in failed_tasks:
            print(f"⚠️  警告：任务 {task['name']} 失败")
            print(f"   错误：{task['last_error']}")
        
        await asyncio.sleep(60)  # 每分钟检查一次
```

### 3. 错误处理

在回调函数中实现完善的错误处理：

```python
async def execute_task(command, metadata):
    try:
        result = await execute(command)
        return result
    except Exception as e:
        # 记录详细错误
        log_error(command, e)
        # 清理资源
        cleanup()
        # 重新抛出异常让管理器处理

```

### 4. 资源管理

确保任务执行后释放资源：

```python
async def execute_task(command, metadata):
    resource = acquire_resource()
    try:
        return await execute(command, resource)
    finally:
        release_resource(resource)
```

## 故障排查

### 任务不执行

1. 检查任务是否启用：`manager.get_task_status("task-name")`
2. 检查 HEARTBEAT.md 文件格式
3. 检查心跳管理器是否启动：`manager.running`

### 任务重复执行

1. 检查 `suppress_duplicates` 配置
2. 检查任务间隔设置
3. 查看执行历史：`manager.get_execution_history()`

### 任务一直失败

1. 查看错误信息：`task_status["last_error"]`
2. 检查执行回调函数
3. 增加重试次数和延迟
